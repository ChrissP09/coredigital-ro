## 4. CSS Variable Fix — Analiza-AI Pages

**Problem:** `tailwind.config.cjs` uses `var(--font-sans)` / `var(--font-mono)` for font families. Astro's Tailwind integration injects CSS using these vars. But analiza-ai pages don't load `main.css` → `tokens.css`, so the vars were undefined.

**Fix:** Inline `<style>` block added to `<head>` in both analiza-ai pages, defining all CSS vars before `app.css` loads.

**Modified:** `src/pages/analiza-ai/index.astro`
**Modified:** `src/pages/analiza-ai/rezultat/[id].astro`

  --font-display:'Inter','Segoe UI','Helvetica Neue',Arial,sans-serif;
  --font-mono:'JetBrains Mono','Fira Code',monospace;
  --text-display:clamp(2.6rem,5vw,4.25rem);
  --text-h1:clamp(2rem,3.5vw,3.25rem);
  --text-h2:clamp(1.75rem,2.5vw,2.25rem);
  --text-h3:clamp(1.15rem,1.5vw,1.5rem);
  --text-h4:1.125rem; --text-body-lg:1.125rem; --text-body:1rem;

  --fw-display:700; --fw-heading:600; --fw-sub:500; --fw-body:400;
